# 客户搜索 / 线索发现 Skill 立项方案

## 基本信息

- Skill 名称：客户搜索 / 线索发现 Skill
- 所属业务节点：主动开发入口 / 客户搜索
- 负责人：Codex 协作起草，后续由半斤九两科技继续推进
- 当前状态：可交付

## 业务问题

- 这个 Skill 要解决的具体问题是什么：
  用户知道目标市场、产品和客户类型，但不会把公开网页搜索变成一套可复制的客户发现流程。
- 当前人工是怎么做的：
  人工在 Google、LinkedIn、目录站和展会站零散搜索，再手工复制公司名、官网和来源链接。
- 为什么值得被做成 Skill：
  它是当前主动开发链路的入口节点，直接决定后续线索整理、客户背调和开发信是否有稳定输入。

## 成功标准

- 最小成功结果是什么：
  给定产品、市场和客户类型，输出一批结构化候选客户名单，带来源链接和补查建议。
- 用户拿到结果后，能直接做什么下一步动作：
  直接桥接进入 `线索整理skill/`。
- 什么情况下算“做出来了但还不能交付”：
  能搜出链接，但没有结构化输出、没有去重、没有来源记录或不能稳定桥接到线索整理时。

## 输入

- 最小输入：
  - `product_or_offer`
  - `target_market`
  - `customer_type`
  - `search_keywords`
- 可选输入：
  - `must_include`
  - `exclude_terms`
  - `max_results`
  - `notes`

## 输出

- 核心输出：
  - 结构化候选名单
  - 来源链接
  - LinkedIn 线索
  - 可见联系人线索
  - 补查建议
  - 线索整理桥接输入
- 输出格式：
  - JSON
  - Markdown

## 核心规则与边界

- 必须遵守的业务规则：
  - 只用公开搜索结果
  - 只做候选发现，不做深度判断
  - 输出必须保留来源链接
- 哪些信息只能写成推断：
  - 联系人身份
  - 公司业务角色
  - 客户价值判断
- 哪些情况必须提示人工复核：
  - 官网或公司名不稳定
  - 只有 LinkedIn 结果没有官网
  - 只有目录页没有明确主体
- 明确不做什么：
  - 不做登录态搜索
  - 不做重型爬虫
  - 不做深度背调
  - 不直接生成开发信

## 验收方式

- 最小测试输入是什么：
  - 食品/冷冻类搜索样例
  - 家纺/家居类搜索样例
- 预期输出应包含哪些关键信息：
  - 结构化候选名单
  - 来源链接
  - 搜索查询回显
  - `follow_up_suggestion`
  - 线索整理桥接输出
- 如何判断这个 Skill 已达到可演示：
  - 能从固定输入稳定产出结构化候选名单
- 如何判断这个 Skill 已达到可交付：
  - 有固定输入结构、固定输出结构、样例输入输出、回归检查、pre-release gate 和最小 OpenClaw 变体
