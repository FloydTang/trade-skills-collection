# 验收与边界

## 当前验收目标

这份组合包当前验收到“最小可演示骨架明确且本地可复现”，不宣称已经达到完整自动化交付。

## 达到可演示的最低条件

- 有完整 README
- 有组合包自己的 `examples/`、`references/`、`scripts/` 结构
- 有固定入口样例
- 有稳定下游 fixture
- 有最小串联脚本
- 能生成阶段化中间产物
- 明确标出人工复核点

当前已完成验证：

- 已实际运行 `scripts/run_minimal_demo.py`
- 已生成完整阶段产物目录
- 已确认组合包文档、样例、脚本之间的引用关系成立

## 达到可交付的补充条件

- 串联脚本可在目标环境重复执行
- 运行依赖说明清楚
- 课程演示路径能直接拿去讲
- 会员交付最小内容边界清楚
- 至少一套输出结果经过复核并可重复生成

## 当前明确不验收的内容

- 总编排器
- 自动发信
- CRM 回写
- 复杂权限系统
- 会员后台
- 完全依赖实时联网的稳定搜索结果

## 当前核心边界

- 组合包只做串联和说明，不重写四个节点的业务逻辑
- 固定样例用于演示和回归，不表示真实市场现状
- 背调和开发信中的推断必须保持保守表达
- 人工复核是设计的一部分，不是首版缺陷
- 当前“可复现”指固定样例模式可复现，不等于实时联网链路稳定可复现
