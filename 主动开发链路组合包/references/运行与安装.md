# 运行与安装

## 这份文件解决什么问题

这份说明只保留当前开源版本怎么最稳地跑起来。

## 当前主入口

当前默认主入口是 `主动开发链路组合包`。

## 当前运行前提

运行这个组合包前，默认满足以下条件：

- 当前工作目录位于母目录 `外贸skill/`
- 这四个节点目录都存在：
  - `客户搜索skill/`
  - `线索整理skill/`
  - `客户背调skill/`
  - `开发信skill/`
- 本地有 `python3`

当前首版不把以下内容当作前提：

- DuckDuckGo 或其他搜索源稳定可用
- 实时联网搜索必须成功
- 外部增强服务已开通

## 当前最稳运行方式

推荐直接在母目录执行：

```bash
python3 ./主动开发链路组合包/scripts/run_minimal_demo.py
```

默认行为：

- 复用 `客户搜索skill/examples/frozen-food-output.json`
- 生成线索整理桥接与筛选结果
- 默认选择 `lead-002`
- 复用组合包自己的稳定背调 fixture
- 输出开发信桥接输入和邮件草稿

## 常用参数

### 1. 指定输出目录

```bash
python3 ./主动开发链路组合包/scripts/run_minimal_demo.py \
  --output-dir ./主动开发链路组合包/outputs/demo-20260325
```

适用场景：

- 留存不同版本演示结果

### 2. 替换邮件阶段的业务信息

```bash
python3 ./主动开发链路组合包/scripts/run_minimal_demo.py \
  --product-or-offer "frozen broccoli and mixed vegetables" \
  --sender-name "Leo" \
  --sender-company "Ningbo FreshGrow Foods"
```

适用场景：

- 保留同一条演示 lead，但换成自己的产品和发件身份

### 3. 手动指定进入背调阶段的 lead

```bash
python3 ./主动开发链路组合包/scripts/run_minimal_demo.py \
  --selected-lead-id lead-002
```

当前限制：

- 当前固定背调 fixture 对应的是 `GreenHarvest Foods`
- 因此默认只建议选 `lead-002`
- 如果改成其他 lead，需要同步替换组合包里的背调 fixture，否则脚本会主动报错

## 推荐检查顺序

第一次跑完后，建议按下面顺序检查：

1. 先看 `03-lead-screening-output.md`
2. 再看 `05-selected-customer-intel-input.json`
3. 再看 `06-customer-intel-report.json`
4. 最后看 `08-email-draft.md`

原因：

- 这四个文件最能解释链路里的判断逻辑和人工复核点

## 常见问题

### 1. 为什么不直接调用实时客户背调脚本

因为当前组合包首版的目标是稳定复现，不是验证实时联网搜索。

### 2. 为什么脚本会限制 lead 与 fixture 对应关系

因为首版强调“固定样例闭环”，不能让上游选中的公司与下游背调报告主体不一致。

### 3. 为什么还要人工复核

因为主动开发本身就包含证据判断、实体匹配和邮件风险控制，人工复核不是缺陷，而是设计的一部分。

### 4. 为什么不是让每个单节点各自创建一套飞书表

增强权益中的数据留存和编排正文不在仓库中展开，请查看飞书文档。
