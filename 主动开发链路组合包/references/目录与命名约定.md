# 目录与命名约定

## 目录约定

组合包首版按下面方式维护：

- `examples/`：组合包自己维护的固定输入和稳定 fixture
- `references/`：运行说明、验收规则、命名说明、样例设计
- `scripts/`：串联脚本和轻量运行入口

## 输出目录建议

建议每次运行都把输出放进单独目录，例如：

- `outputs/demo-20260325/`
- `outputs/demo-20260407/`

## 阶段编号规则

建议所有中间产物统一使用两位阶段编号：

- `01-`
- `02-`
- `03-`
- `04-`
- `05-`
- `06-`
- `07-`
- `08-`

## 推荐文件名

- `01-lead-discovery-output.json`
- `02-lead-screening-input.json`
- `03-lead-screening-output.json`
- `03-lead-screening-output.md`
- `04-customer-intel-batch.json`
- `05-selected-customer-intel-input.json`
- `06-customer-intel-report.json`
- `07-email-input.json`
- `08-email-draft.json`
- `08-email-draft.md`

## 命名原则

- 文件名优先表达阶段和用途
- 不把“临时、最终、最新版”写进文件名
- 同一个阶段的 JSON 和 Markdown 尽量同名前缀
- 对固定样例和实际运行都保持同一套命名
