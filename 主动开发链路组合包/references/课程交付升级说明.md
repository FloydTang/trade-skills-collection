# 课程交付升级说明

这份文档只服务课程交付口径，不进入 runtime 契约。

## 一、飞书的定位

统一口径：

- 飞书是课堂标准沙盘，不是企业唯一答案
- 企业已有 CRM、ERP、Excel、Google Sheet、WPS、企业微信表格，都可以是数据容器
- 课堂先跑通标准数据契约和闭环，不要求所有人现场接企业系统

## 二、课堂主线

新的主线不是“会不会搭飞书”，而是：

- 有没有跑通标准字段
- 有没有形成可复核的主动开发闭环
- 有没有开始沉淀客户开发数据资产

## 三、建议教学顺序

教学顺序建议调整为：

1. 先用客户背调建立信任
2. 再讲主动搜索的边界和证据分级
3. 再讲线索整理和质量门槛
4. 最后讲复核型开发信工作台和数据回流

## 四、课前 48 小时预检

建议收集：

- 主营产品
- 目标国家/市场
- 是否已有 CRM
- 使用 Gmail / Outlook / 企业邮箱
- 是否愿意用飞书
- 是否有技术同事
- 一个已知客户官网

## 五、课程与课后承接

### 线下课

- 跑通标准闭环
- 完成第一个真实案例
- 看懂数据契约和容器方案

### 1 个月复盘

- 解决部署和 Skill 卡点
- 解决表格 / 飞书 / 字段映射问题

### 年度会员

- 持续更新 Skill、SOP、工具和接入案例

### 高阶定制

- CRM / ERP / 邮箱 / 私有系统接入
