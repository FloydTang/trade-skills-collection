# Skill 验收清单

在 Skill 进入“可演示”或“可交付”前，逐项检查。

## 基础可用性

- [ ] 能从最小输入跑通
- [ ] 输入格式清楚，不依赖口头补充理解
- [ ] 输出是结构化结果，不是零散文本堆砌
- [ ] 有最少一个示例输入
- [ ] 有清楚的使用说明

## 结果质量

- [ ] 明确区分事实、证据和推断
- [ ] 没有伪造事实或过度脑补
- [ ] 默认行为和异常情况说明清楚
- [ ] 输出内容能直接支持下游业务动作

## 协作适配

- [ ] 至少适合龙虾 / OpenClaw 或 Codex 其中一种调用方式
- [ ] 目录结构清楚，后续可以继续改造
- [ ] 关键脚本、模板或规则有对应文档

## 课程与内容价值

- [ ] 能讲成课程案例
- [ ] 能说明为什么这样设计
- [ ] 能解释输入、输出、边界和业务价值

## 发布判断

- [ ] 达到“可演示”标准
- [ ] 达到“可交付”标准
- [ ] 如果仍不达标，已记录当前缺口和下一步动作
