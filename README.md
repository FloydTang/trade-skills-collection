# 外贸 Skill 母目录

这个目录用于统一孵化、维护和沉淀适合外贸业务各节点的实用型 Skill。

它不是单一 Skill 仓库，而是一个面向业务结果、课程表达、发布复用和长期演进的 Skill 合集仓库。这个仓库默认承担“分发与演示仓库”角色，当前优先服务两类协作对象：

- 龙虾 / OpenClaw：偏云端调用、自动化、多代理协作
- Codex：偏本地开发、调试、文档沉淀和二次改造

## 发布结构

这个母目录默认采用“三层结构”推进，而不是一开始就做一个大而全的总 Skill。

### 第一层：单节点 Skill

- 每个 `*skill/` 子目录默认先做成一个独立发布单元
- 它既是业务中的最小可用节点，也是课堂上最适合即插即用的安装单元
- 当前主发布节奏以这一层为准

### 第二层：链路组合包

- 当多个单节点 Skill 的输入输出稳定后，再把它们按业务顺序组合成轻量链路
- 这一层主要服务课程演示、业务闭环验证和安装复用
- 它不是新的重型 Skill，而是对已有节点 Skill 的组合使用

### 第三层：总 Skill / 总编排器

- 总 Skill 放在后续阶段推进
- 它的职责是调度和串联各节点 Skill，而不是替代各节点 Skill 的独立存在
- 只有当前两层稳定后，才进入这一层

### 版本策略

- Codex 版与 OpenClaw / 龙虾版属于运行形态差异
- 它们不改变“三层结构”的发布策略
- 默认先让每个节点 Skill 可单独安装、单独演示、单独交付
- 整体打包和总编排器放到后期统一推进

## 目录角色

- `AGENTS.md`：总协作手册，定义这里为什么存在、怎么协作、什么样的 Skill 才算合格，以及如何按“三层结构”推进
- `skill需求池.md`：统一维护 Skill 候选方向、状态和下一步动作
- `模板/`：新 Skill 立项和验收时复用的模板
- `客户背调skill/`：首个已落地样板 Skill，当前作为母目录中的基准节点继续保留，并在合集仓库中提供完整目录副本
- 其他 `*skill/` 子目录：首批重点方向的占位目录，用于后续逐步立项和开发

## 当前进度

- `客户背调skill/`：已作为首个样板 Skill 落地，当前在母目录中保留其目录、脚本和说明，用于承接后续链路衔接与课程表达
- `开发信skill/`：已达到“可交付”，具备本地版脚本、schema 校验、样例输入输出、回归检查、pre-release gate 和最小 `for-openclaw/` 变体
- `skill需求池.md`：已明确主动开发推荐链路和当前优先级
- `模板/`：已用于后续单节点 Skill 的立项和验收

## 样板 Skill 与独立仓库

`客户背调skill/` 在这个母目录里承担两层角色：

- 作为“主动开发链路”中的已落地样板节点，供后续 `线索整理 / 初筛 -> 客户背调 -> 开发信` 衔接使用
- 作为一个已经独立发布过的方法论样板，对应 GitHub 仓库：`https://github.com/FloydTang/trade-customer-intel`

这意味着母目录不会删除 `客户背调skill/`，而是继续保留其在链路中的位置说明；独立仓库继续承担对外单 Skill 发布和演进，合集仓库则保留一个面向分发与演示的稳定副本。

`开发信skill/` 也采用相同逻辑：

- 独立仓库：`https://github.com/FloydTang/trade-outreach-email`
- 合集仓库：保留当前阶段的稳定副本，便于用户一键下载与课程演示

## 当前推送策略

- 母目录作为统一框架持续维护和持续推送
- 每完成一个达到“可演示”或“可交付”的单节点 Skill，就增量同步到母目录仓库
- 不等待所有 Skill 完成后再集中发布
- 独立仓库已存在的样板 Skill，保留其独立发布节奏；母目录负责链路定位、目录协同和后续组合说明
- 后续子 Skill 默认参考 `客户背调skill/` 的逻辑：单节点 Skill 可独立发布，合集仓库保留阶段性稳定副本，方便用户一键下载

## 双轨仓库逻辑

- 单个 Skill 独立仓库：作为该 Skill 的主开发源
- Skill 合集仓库：作为课程演示、统一分发和一键下载入口
- 独立仓库可更快迭代
- 合集仓库只收录已经达到“可演示”或“可交付”的稳定版本
- 不是每一次微调都必须立刻同步合集仓库

## 合集收录标准

准备进入合集仓库的节点 Skill，至少需要满足：

- 有 `README.md`
- 有 `SKILL.md`
- 有最小示例输入
- 有明确输入输出结构
- 有默认行为、核心规则和风险边界
- 能从最小样例跑通
- 目录结构清楚
- 如涉及链路衔接，需写明与上下游节点的关系
- 明确标注当前状态与作者

## 使用方式

如果要新增一个 Skill，按这个顺序操作：

1. 先读 `AGENTS.md`
2. 在 `skill需求池.md` 中完成立项
3. 复制 `模板/skill立项模板.md`
4. 确认边界、输入输出和验收标准
5. 再进入对应子目录开始实现

## 当前原则

- 优先做即插即用的单节点业务 Skill，不先追求复杂系统
- 优先服务真实客户问题、课程高频问题和内容反馈
- 优先先发布单节点，再组织链路组合，最后才做总编排器
- 输出必须可复用、可验证、可继续改造
- 默认中文协作，对外销售内容允许中英混合
