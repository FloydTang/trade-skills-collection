# 外贸 Skill 需求池

这份文档用于统一维护外贸业务节点 Skill 的候选方向、状态和下一步动作。

使用规则：

- 新 Skill 必须先在这里立项，再进入开发
- 条目要写成业务化需求，不写成课程标题复述
- 每次状态变化，都要更新“当前状态”和“下一步动作”

## 当前发布策略说明

这个母目录当前采用“三层结构”推进：

1. 单节点 Skill：当前主发布单位，也是课堂上即插即用的最小单元
2. 链路组合包：当多个节点稳定后，用于课程演示、业务闭环验证和轻量自动化复用
3. 总 Skill / 总编排器：放在后续阶段，负责调度已有节点 Skill，不进入当前单节点发布节奏

当前推荐主动开发链路为：

`客户搜索 / 线索发现 -> 线索整理 / 初筛 -> 客户背调 -> 开发信 -> 跟进优先级 / 客户管理`

当前优先策略：

- 先把单节点 Skill 做到可演示或可交付
- 再逐步组织链路组合包
- 总编排器作为后续阶段，不单独立项为当前主线条目

## 当前优先级说明

- 第一优先级：能直接服务“主动开发闭环”演示和培训增长的方向
- 第二优先级：能补齐业务闭环、但暂不急于开发的方向
- 被动获客相关方向暂不删除，但降为后续扩展
- 当前优先锁定首批 5 个主方向，避免范围失控

## 1. 客户搜索 / 线索发现 Skill

- Skill 名称：客户搜索 / 线索发现 Skill
- 所属业务节点：主动开发入口 / 客户搜索
- 用户痛点：知道大概要找哪类客户，但不会系统搜，不会快速从公开网页和 LinkedIn 找出第一批潜在客户
- 典型输入：目标市场、产品、客户类型、关键词、补充限制条件
- 目标输出：结构化候选客户名单，包含公司名、来源链接、可见联系人线索、补查建议
- 适用对象：外贸业务员、销售主管、老板、课程学员
- 业务价值：把“手工搜客户”升级成“可复制的客户发现流程”，帮助企业更快建立主动开发起点
- 对应课程/内容来源：外贸 AI 落地营中的主动开发样板流程、AI 搜客户方法表达
- 需要的能力模块：网页搜索、LinkedIn 搜索、结果去重、基础字段抽取、来源记录
- 风险与边界：只用公开信息；不承诺完整名单；不做大规模爬取；优先通用轻量路线
- 当前状态：候选
- 下一步动作：定义最小输入字段和最小输出结构，做一版样板演示流程，作为主动开发链路入口节点的单节点候选

## 2. 线索整理 / 初筛 Skill

- Skill 名称：线索整理 / 初筛 Skill
- 所属业务节点：主动开发中间层 / 名单整理
- 用户痛点：搜到一堆公司名、链接和零散联系人后，信息太乱，无法快速变成可判断、可跟进的线索池
- 典型输入：候选客户名单、公司链接、联系人线索、备注
- 目标输出：统一字段后的线索池，包含缺失项、初步分类、可补查建议
- 适用对象：外贸业务员、销售支持、课程学员
- 业务价值：把零散搜索结果整理成能进入背调和开发的标准输入，减少后续判断成本
- 对应课程/内容来源：外贸 AI 落地营中的线索整理 / 初筛逻辑
- 需要的能力模块：字段规范化、缺失检测、初步分类、优先级提示、导出能力
- 风险与边界：初筛只是辅助，不替代人工判断；不做复杂 CRM；首版以轻量结构为主
- 当前状态：候选
- 下一步动作：先定义最小字段集，和客户背调 Skill 输入结构保持兼容，作为主动开发链路中间层的单节点候选

## 3. 客户背调 Skill

- Skill 名称：客户背调 Skill
- 所属业务节点：主动开发判断中枢 / 客户开发前置研究
- 用户痛点：拿到公司名、邮箱或联系人后，不知道客户真假、官网真假、联系人真假，也无法快速整理成销售可用判断
- 典型输入：公司名、联系人名、邮箱、官网、国家市场、补充备注
- 目标输出：结构化客户情报报告，包含身份快照、数字足迹、风险评级、销售切入角度、可选英文触达草稿
- 适用对象：外贸业务员、销售支持、老板、课程学员
- 业务价值：降低主动开发中的盲目投入，把零散搜索升级成证据驱动判断，为首轮触达提供依据
- 对应课程/内容来源：外贸专区第1课《客户背调 Skill》、AI编程实操课第8课、第12课、外贸 AI 落地案例内容
- 需要的能力模块：网页搜索、页面抓取、实体匹配、证据分级、风险判断、报告生成
- 风险与边界：只使用公开信息；弱匹配必须标注不确定；不做法律或 KYC 级尽调；不捏造个性化内容
- 当前状态：可演示
- 下一步动作：保留为母目录样板 Skill；作为单节点可演示节点继续维护，并补充其在“搜索 / 初筛 / 开发信”组合链路中的衔接说明；独立仓库对应 `https://github.com/FloydTang/trade-customer-intel`，合集仓库保留稳定副本

## 4. 开发信 Skill

- Skill 名称：开发信 Skill
- 所属业务节点：主动开发触达 / 首轮开发 / 报价与跟进
- 用户痛点：开发信、报价邮件和跟进邮件长期依赖人工拼凑，效率低、针对性弱、风格不稳定
- 典型输入：客户基础信息、客户画像摘要、产品信息、报价范围、跟进阶段、目标语气
- 目标输出：可直接修改发送的英文开发信、报价邮件、跟进邮件，支持多个版本或风格
- 适用对象：外贸业务员、销售支持、课程学员
- 业务价值：缩短主动开发中的触达准备时间，提升触达一致性，把背调结果转成实际销售动作
- 对应课程/内容来源：AI编程实操课第10课《外贸邮件自动生成器》、外贸内容中关于客户开发与转化的话题
- 需要的能力模块：模板管理、变量填充、风格控制、版本生成、输入标准化
- 风险与边界：不捏造客户需求；不把未确认事实写进邮件；不直接代替人工审核和发送
- 当前状态：可交付
- 下一步动作：保留为主动开发链路中的单节点可交付节点，进入合集仓库增量推送，并按与客户背调 Skill 相同的双轨逻辑维护独立仓库；独立仓库对应 `https://github.com/FloydTang/trade-outreach-email`；后续围绕 `follow_up` 模板质量、CI 接入和 OpenClaw 实战反馈继续增强

## 5. 跟进优先级 / 客户管理 Skill

- Skill 名称：跟进优先级 / 客户管理 Skill
- 所属业务节点：主动开发后续承接 / CRM 轻量管理
- 用户痛点：主动开发出来的客户多了之后，不知道今天先跟谁，跟进记录又容易分散和漏掉
- 典型输入：客户基础资料、最近互动时间、风险等级、阶段标签、下次动作、备注
- 目标输出：按优先级排序的待跟进清单，或结构化客户档案与基础提醒结果
- 适用对象：外贸业务员、销售支持、小团队管理者、课程学员
- 业务价值：把主动开发出来的客户线索沉淀下来，减少漏跟进和低价值消耗
- 对应课程/内容来源：外贸真实业务动作、客户背调与开发信之后的承接需求、AI 工作流课程中的任务排序思路
- 需要的能力模块：状态更新、排序规则、搜索过滤、提醒逻辑、导出能力
- 风险与边界：默认轻量本地版本，不先做复杂数据库；排序只作辅助决策，不替代业务判断
- 当前状态：候选
- 下一步动作：先拆成“跟进优先级”与“客户管理”两条轻量路线，再决定首版先做哪个，并明确其作为主动开发链路后续承接节点的单节点发布方式

## 6. 竞品监控 Skill

- Skill 名称：竞品监控 Skill
- 所属业务节点：市场研究 / 竞品跟踪
- 用户痛点：竞品官网、价格、新品和动态需要反复人工查看，耗时且难持续
- 典型输入：竞品网址列表、监控频率、关注字段、通知方式
- 目标输出：竞品变化汇总报告，突出价格变化、上新动态、促销活动和重点异常
- 适用对象：老板、运营、销售支持、市场研究人员、课程学员
- 业务价值：提高市场反应速度，帮助及时调整报价、选品和客户沟通策略
- 对应课程/内容来源：AI编程实操课第9课《竞品数据监控工具》、AI+外贸研究和市场洞察内容
- 需要的能力模块：定时抓取、结构化抽取、历史对比、异常标注、报告通知
- 风险与边界：遵守目标网站访问边界；不承诺全站稳定抓取；网站结构变化需要维护
- 当前状态：候选
- 下一步动作：先确定最小监控对象和报告字段，避免一开始做成重型系统
