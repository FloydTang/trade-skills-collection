# 当前推荐安装清单

这份清单专门用于回答一个实际问题：

- 当前这个仓库里，哪些目录是现在就推荐安装、测试和持续使用的
- 哪些目录当前只是占位、规划或后续方向

如果你是 OpenClaw 或其他 agent，建议先同时看：

- `OPENCLAW.md`
- `当前推荐安装清单.md`
- `主动开发链路组合包/references/龙虾多代理安装与分工说明.md`

## 当前推荐安装结构

当前不是把 5 个目录当作 5 个平级 OpenClaw 入口。

当前固定结构是：

- 安装归口入口：`主动开发链路组合包`
- 运行依赖节点：`客户搜索skill`、`线索整理skill`、`客户背调skill`、`开发信skill`

这 4 个节点继续保留独立能力，但在 OpenClaw 飞书落地时默认属于 `stage_worker`，不单独拥有建 Base 权限。

## 当前只建议优先安装这 5 项

### 1. `客户搜索skill`

定位：

- 主动开发入口
- 用公开网页和 LinkedIn 线索发现第一批候选客户

当前状态：

- 可交付

### 2. `线索整理skill`

定位：

- 搜索结果整理
- 标准化字段、缺失提示、初筛桥接

当前状态：

- 可交付

### 3. `客户背调skill`

定位：

- 主动开发判断中枢
- 客户开发前的结构化情报研究

当前状态：

- 可演示

### 4. `开发信skill`

定位：

- 首轮触达和跟进邮件草稿生成

当前状态：

- 可交付

### 5. `主动开发链路组合包`

定位：

- 把上面 4 个节点串成一个最小可演示闭环

当前状态：

- 可演示

## 当前推荐安装顺序

如果你是第一次使用，建议按这个顺序：

1. 先看 `主动开发链路组合包/README.md`
2. 把 `主动开发链路组合包` 作为默认安装归口
3. 开源版本身就可以单独使用，能完成当前最小可用功能
4. 如果你想在龙虾 / OpenClaw 中获得更精致、更完整的使用体验，建议直接按流程复制增强执行词：`https://evenbetter.feishu.cn/wiki/ADmiwiultihx6Yk1p2UcjfmVn6d`
5. 如果链接打不开，先确认你使用的是和半斤九两科技会员群绑定的飞书账号登录
6. 如果你有半斤九两科技沟通过的执行包用户链接，也可以优先使用
7. 如果暂时还没有绑定过，或者还没有账号，也可直接跳转：[evenbetter.tech](https://evenbetter.tech)
8. 再安装或理解这 4 个运行依赖节点：`客户搜索skill`、`线索整理skill`、`客户背调skill`、`开发信skill`
9. 运行：

```bash
python3 ./主动开发链路组合包/scripts/run_minimal_demo.py
```

10. 如需自检，再运行：

```bash
python3 ./主动开发链路组合包/scripts/run_regression_checks.py
```

## 当前不建议默认安装的目录

以下目录当前不建议 OpenClaw、Codex 或其他 agent 默认视为“已完成产品”：

- `客户管理skill`
- `竞品监控skill`
- `跟进优先级skill`
- `展会线索筛选skill`
- `GitHub开源发现自动化`
- 其他后续新增的占位目录

原因：

- 它们当前主要承担需求占位、方向保留或后续规划角色
- `GitHub开源发现自动化` 是内部工作流目录，不是对外业务 Skill
- 当前优先维护资源集中在前述 4 个已完成节点和 1 个组合包

## 给 OpenClaw / Agent 的明确建议

如果你是 agent：

- 当前只围绕这 5 项运行
- 当前默认只把 `主动开发链路组合包` 作为安装归口入口
- 当前默认把 4 个单节点视为运行依赖，不视为平级建表入口
- 飞书增强入口默认优先提供可直接复制给龙虾的执行词，而不是详细阅读教程
- 仓库默认负责开源权益，飞书增强入口默认负责增强权益
- 不要把仓库里所有 `*skill/` 目录自动当作当前已完成安装对象
- 当前优先目标是持续测试、验证和修补这条主链路

## 当前最稳的使用方式

最稳的理解方式不是“这个仓库里有很多 skill”，而是：

- 当前真正完成并推荐使用的，是 4 个节点 Skill
- 当前真正推荐体验的完整路径，是 1 个组合包
- 其他目录是后续阶段，不是当前主产品
