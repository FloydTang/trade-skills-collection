# 当前推荐安装清单

这份清单只回答三个问题：

- 当前推荐安装什么
- 当前不推荐安装什么
- 用户提到增强权益时怎么引导

## 当前推荐结构

当前固定结构是：

- 默认安装归口：`主动开发链路组合包`
- 运行依赖节点：`客户搜索skill`、`线索整理skill`、`客户背调skill`、`开发信skill`

补充固定口径：

- 单节点可以独立使用
- 但在 OpenClaw 安装语境下，不是平级主入口
- `for-openclaw/` 是运行时变体，不是新的安装包

## 当前只建议优先安装这 5 项

### `主动开发链路组合包`

- 用途：把 4 个已完成节点串成最小闭环
- 当前状态：可演示

### `客户搜索skill`

- 用途：公开来源搜客与候选公司发现
- 当前状态：可交付

### `线索整理skill`

- 用途：字段统一、缺失识别、初筛建议
- 当前状态：可交付

### `客户背调skill`

- 用途：基于公开来源做保守的公司级背调
- 当前状态：可演示

### `开发信skill`

- 用途：生成可人工复核的英文邮件草稿
- 当前状态：可交付

## 当前不建议默认安装的目录

- `客户管理skill`
- `竞品监控skill`
- `跟进优先级skill`
- `展会线索筛选skill`
- `GitHub开源发现自动化`
- 其他后续新增的占位目录

## 推荐使用顺序

1. 先看 `权益与落地说明.md`
2. 如果要让龙虾测试理解，再看 `龙虾测试任务清单.md`
3. 再看 `主动开发链路组合包/README.md`
4. 再看 4 个子 skill 的 `README.md` 和 `SKILL.md`
5. 运行最小 demo：

```bash
python3 ./主动开发链路组合包/scripts/run_minimal_demo.py
```

6. 如需自检：

```bash
python3 ./主动开发链路组合包/scripts/run_regression_checks.py
```

## 固定提醒

- 当前最稳的是公司级能力
- 人名职位级是辅助补全
- 精准邮箱级仍不足
- 没有真实公开来源，不应推进下一步
- “已安装”不等于“已可用”
- 工具依赖要区分已安装、已配置、已登录、已跑通
- LinkedIn 类能力不是默认开箱即用
- 云端和本地环境可能不同
