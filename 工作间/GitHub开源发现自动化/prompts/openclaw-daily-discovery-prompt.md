# 龙虾每日 GitHub 开源发现任务提示词

请执行一个“GitHub 开源工具 / Skill 每日发现与评审”任务，服务当前外贸 Skill 主路线的持续增强。

你的目标不是做泛 AI 工具榜单，而是每天从 GitHub 里找出少量真正值得我们关注的开源工具、Skill、脚本或工作流模板，并把结果沉淀到固定文档。

## 一、工作边界

只围绕以下方向搜索：

- 客户搜索 / lead discovery
- 线索整理 / lead screening
- 客户背调 / customer intel
- 开发信 / outreach email
- 跟进优先级 / lightweight CRM / follow-up workflow
- 竞品监控 / website monitoring / change detection
- OpenClaw / agent workflow / promptable skill packaging / Markdown-first automation

不要扩展成泛 AI 产品收集、泛销售软件收集或无关的 GitHub 热门仓库推荐。

## 二、每日动作

请每天执行以下动作：

1. 在 GitHub 上搜索与上述方向直接相关的开源项目
2. 初步去重，排除明显无关、长期不维护、文档极弱或许可证不清晰的项目
3. 当天只精选 3-5 个候选
4. 对每个候选给出结构化评价和明确结论标签
5. 更新当日日报
6. 更新累计观察池
7. 更新落地候选清单
8. 如已有明确人工结论，再更新决策记录

## 三、必须使用的文件

请使用并维护以下文件：

- 当日日报：`工作间/GitHub开源发现自动化/reports/YYYY-MM-DD.md`
- 观察池：`工作间/GitHub开源发现自动化/github-watchlist.md`
- 落地候选清单：`工作间/GitHub开源发现自动化/tool-adoption-backlog.md`
- 决策记录：`工作间/GitHub开源发现自动化/tool-adoption-decisions.md`

如果当日日报文件不存在，请按目录中的模板结构创建。

## 四、每个候选必须输出的字段

对每个精选候选，至少输出：

- 项目名称
- GitHub 链接
- 一句话定位
- 对应我们的哪条业务路线或哪个现有 Skill
- 它更像：可直接复用工具 / 可借鉴模块 / 可借鉴工作流 / 仅供参考
- 许可证情况
- Star、最近活跃情况、最近更新时间
- 文档和样例是否完整
- 接入成本判断：低 / 中 / 高
- 相关性评分：1-5
- 课程表达价值：1-5
- 推荐动作：建议纳入工具库 / 建议进入需求池 / 建议继续观察 / 不建议处理
- 推荐原因
- 风险与不确定点
- 是否需要人工复核

## 五、筛选标准

优先保留：

- 与当前外贸 Skill 主路线直接相关
- 能增强现有节点 Skill，而不是平行替代一个大系统
- 开源许可证清晰
- 最近仍有维护，或至少不是明显废弃
- 有 README、样例或明确输入输出
- 具备课程表达价值
- 接入成本可控，或方法价值明显高

优先排除：

- 大而全但和当前路线衔接弱
- 长期停更、文档薄弱、样例缺失
- 许可证风险不清楚
- 需要重型基础设施改造才有意义
- 更像行业新闻，而不是可落地工具资产

## 六、固定结论标签

每个候选必须只选一个标签：

- `建议纳入工具库`
- `建议进入需求池`
- `建议继续观察`
- `不建议处理`

## 七、人工复核边界

以下情况必须标记“需要人工复核”，不能直接判为 `建议纳入工具库`：

- 许可证不清晰
- 仓库活跃度异常
- 与当前主路线关系不够直接
- 接入成本明显偏高
- 需要改动我们现有架构或发布策略

## 八、日报要求

当日日报必须固定包含这些部分：

- 今日结论摘要
- 今日精选 3-5 个候选
- 每个候选的结构化评价
- 今日不推荐处理项目简表
- Watchlist 状态更新
- 建议今天优先人工判断的 1-3 个项目
- 如果今天没有高质量新发现，要明确写“今天无值得推进的新候选”

## 九、最终目标

这项任务的目标，是让我们的 Skill 生产逐步从“完全依赖内部从零开发”，升级成“持续发现优质开源基础并在其上迭代”。

重点是服务当前外贸 Skill 规划路线，而不是做收藏夹。
