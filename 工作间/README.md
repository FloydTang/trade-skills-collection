# 工作间

这个目录是当前 `外贸skill` 分类下，GitHub 开源发现项目的长期记录层。

它对应的是 Obsidian `工具工作间/外贸skill`，不是 Obsidian 顶层“工具工作间”本身。

这里的长期记录一旦稳定，应在 Obsidian `工具工作间/外贸skill` 里有对应展现。展现形式可以是镜像、摘要、索引或说明，不要求复制执行层文件。

这里只保留会被反复复用、反复判断、反复协作的有效内容。

顶级原则固定为：

`实事求是、简洁高效`

## 只保留什么

- 已经看过并值得继续保留判断的开源项目
- 这些项目与当前外贸 Skill 路线的映射关系
- 是否进入 `skill需求池.md` 的明确建议

## 不保留什么

- 每天搜索过程的细节噪音
- 为了留痕而留痕的中间表
- 与后续判断无关的大段日报搬运

## 当前最小结构

```text
工作间/
├── README.md
├── 开源工具总表.md
└── 路线映射.md
```

如某个方向已经进入持续推进、且后续会被课程、品牌、产品表达反复引用，可以补充少量“推进说明”类文档，但仍要遵守一个原则：

- 只补长期会复用的人类阅读文档
- 不把执行过程噪音搬进来

## 维护规则

- `开源工具总表.md` 是唯一主表
- `路线映射.md` 只负责回答“怎么用”
- `skill需求池.md` 只接收已经形成明确方向判断的内容
- 如果一个信息未来不会被再次引用，就不要写进这里
- 如果一个记录未来会被课程、品牌、产品表达反复引用，就应同步到 Obsidian 对应文件夹可见

`GitHub开源发现自动化/reports/每日日报`

-> `工作间/路线映射.md`
-> `skill需求池.md`
