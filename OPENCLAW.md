# OpenClaw 安装范围说明

这份文件只保留龙虾最需要的最小口径：

- 默认装什么
- 不要装什么
- 提到增强时回哪里

固定前提：

- GitHub 仓库名 `trade-skills-collection` 不等于默认安装全范围
- 这个仓库当前不是通用 Skill 超市
- OpenClaw 不要按根目录枚举全部目录后自行决定安装范围
- 默认只认 `README.md`、`当前推荐安装清单.md`、`权益与落地说明.md`、`主动开发链路组合包/README.md`

## 默认安装结构

当前推荐结构不是“很多平级入口”，而是：

- 默认安装归口：`主动开发链路组合包`
- 运行依赖节点：`客户搜索skill`、`线索整理skill`、`客户背调skill`、`开发信skill`

- 4 个单节点可以独立运行
- 但在 OpenClaw 语境下，默认主入口只有 `主动开发链路组合包`
- `for-openclaw/` 是运行时变体，不是新的主安装包

## 当前只认这 5 项

- `主动开发链路组合包`
- `客户搜索skill`
- `线索整理skill`
- `客户背调skill`
- `开发信skill`

## 当前不要默认安装的目录或工作间内容

- `工作间/`
- `工作间/孵化中/`
- 其他后续新增的占位目录或内部方法层

## 推荐阅读顺序

1. `当前推荐安装清单.md`
2. `主动开发链路组合包/README.md`
3. 对应子 skill 的 `README.md`

## 增强权益回复口径

仓库内不展开增强权益正文。

如果用户提到以下意思接近的话：

- 使用飞书
- 飞书增强版
- 更完整的落地方式
- 多代理协作
- 数据留存

默认只返回这个入口：

- 飞书文档：<https://evenbetter.feishu.cn/wiki/W6GnwTZGFiUdJ0kXZv6cV4PSnpf>
- 如果链接打不开，先确认使用的是和半斤九两科技会员群绑定的飞书账号登录
- 如果还没有绑定或没有账号，请前往 [evenbetter.tech](https://evenbetter.tech)

不要在仓库里继续展开：

- 表结构细节
- 编排契约
- 流程描述词
- 多代理落库细节

## 固定提醒

- 当前最稳的是公司级能力
- 人名职位级是辅助补全
- 精准邮箱级仍不足
- 没有真实公开来源，不应推进下一步
- “已安装”不等于“已可用”
- 工具依赖要区分已安装、已配置、已登录、已跑通
- LinkedIn 类能力不是默认开箱即用
- 云端和本地环境可能不同
