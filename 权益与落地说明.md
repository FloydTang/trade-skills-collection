# 权益与落地说明

这份文件只保留一个判断：

- 开源看仓库，增强看飞书

先固定一条判断：

- GitHub 仓库名 `trade-skills-collection` 不等于当前公开安装全范围
- 当前仓库公开交付的是“外贸主动开发主线”
- 不是通用 Skill 合集，更不是根目录全量安装清单

## 开源权益

开源权益当前完整承载在仓库内。

开源权益包括：

- `主动开发链路组合包` 这个默认主入口
- `客户搜索skill`
- `线索整理skill`
- `客户背调skill`
- `开发信skill`

除此之外的目录，当前都不应被当成默认安装对象。

开源权益当前最准确定位：

- 公司级可用
- 人名职位级基本可用
- 精准邮箱级仍不足

固定提醒：

- 没有真实公开来源，不应推进下一步
- `已安装` 不等于 `已可用`
- 工具依赖要区分 `已安装`、`已配置`、`已登录`、`已跑通`
- LinkedIn 类能力不是默认开箱即用
- 云端和本地环境可能不同

## 增强权益

增强权益不在仓库中展开正文。

增强权益用于承接：

- 数据留存
- 统一编排
- 多代理协作
- 飞书课堂沙盘与文档落地

如需增强权益，请查看飞书文档：

- <https://evenbetter.feishu.cn/wiki/W6GnwTZGFiUdJ0kXZv6cV4PSnpf>

## 推荐路径

建议按这个顺序：

1. 先看仓库根目录 `README.md`
2. 再看 `主动开发链路组合包/README.md`
3. 先跑 `主动开发链路组合包/scripts/run_minimal_demo.py`
4. 再按需进入对应子 skill
5. 如需数据留存和协作编排，再进入飞书文档

## 固定口径

- 仓库负责开源权益说明
- 飞书文档负责增强权益说明
- 不要根据仓库中的历史残留文件推断增强权益正文
- 不要根据根目录里的其他目录推断当前公开安装范围
- 标准数据契约优先于具体容器
- 飞书不是企业唯一数据容器
