# 线索整理 / 初筛 Skill 立项方案

## 基本信息

- Skill 名称：线索整理 / 初筛 Skill
- 所属业务节点：主动开发中间层 / 名单整理
- 负责人：Codex 协作起草，后续由半斤九两科技继续推进
- 当前状态：设计中

## 业务问题

- 这个 Skill 要解决的具体问题是什么：
  搜索阶段通常会产出零散公司名、官网链接、LinkedIn 结果、联系人名和备注，但这些信息格式不统一、缺失项多，无法直接进入客户背调或继续跟进。
- 当前人工是怎么做的：
  人工复制粘贴到 Excel、表格或聊天记录里，再手工补字段、删重复、标记可疑项，最后挑出少量值得继续查的线索。
- 为什么值得被做成 Skill：
  它是“客户搜索”到“客户背调”之间最容易断掉的一环。只要这一层标准化，就能把后续背调、开发信和客户管理串起来。

## 成功标准

- 最小成功结果是什么：
  给定一份零散候选线索，输出一份字段统一、缺失项明确、可直接进入客户背调的结构化线索池。
- 用户拿到结果后，能直接做什么下一步动作：
  直接挑选线索进入 `客户背调skill/`，或先补查缺失字段后再进入背调。
- 什么情况下算“做出来了但还不能交付”：
  能整理字段，但缺少明确分类规则、缺失提示、输入示例或与客户背调的兼容说明时，只能算可演示前阶段，不算可交付。

## 输入

- 最小输入：
  一条或多条候选线索，每条至少包含以下任一项：
  - `company_name`
  - `company_website`
  - `email`
  - `person_name`
- 可选输入：
  - `source_url`
  - `linkedin_url`
  - `country_or_market`
  - `notes`
  - `product_keywords`
  - `source_type`
- 输入示例：

```json
[
  {
    "company_name": "Nordic Home Textile AB",
    "company_website": "https://www.nordichometextile.example",
    "person_name": "Nadia",
    "email": "",
    "country_or_market": "Sweden",
    "source_url": "https://www.linkedin.com/company/nordic-home-textile-ab",
    "notes": "Found via home textile search results"
  },
  {
    "company_name": "",
    "company_website": "",
    "person_name": "",
    "email": "purchasing@acme-industrial.com",
    "country_or_market": "United States",
    "source_url": "https://example.com/acme",
    "notes": "Email collected from directory page"
  }
]
```

## 输出

- 核心输出：
  - 统一字段后的线索池
  - 每条线索的缺失字段提示
  - 初步分类结果
  - 是否建议进入客户背调
  - 给客户背调 Skill 使用的桥接字段

## 核心规则与边界

- 必须遵守的业务规则：
  - 首版只做字段整理和初筛提示，不伪造缺失字段
  - 输出必须明确哪些字段来自原始输入，哪些只是规范化结果
  - 与客户背调 Skill 对接时，字段命名尽量保持兼容
- 哪些信息只能写成推断：
  - 公司类型、岗位角色、采购意向、客户价值等级
  - 任何基于域名、来源页面或名称相似度得出的判断
- 哪些情况必须提示人工复核：
  - 同一条线索出现多个可能官网
  - 公司名与域名明显不一致
  - 只有邮箱没有公司名且域名不清晰
  - 联系人与公司之间关联很弱
- 明确不做什么：
  - 不直接做复杂打分模型
  - 不替代客户背调
  - 不做 CRM 长期存储
  - 不把“可继续查”写成“高价值客户”

## 依赖与实现方式

- 需要的脚本 / 工具 / API：
  - 首版优先本地 Python 脚本
  - 读写 JSON / CSV
  - 基础字段规范化逻辑
- 是否需要联网：
  - 首版默认不需要联网
  - 只处理已有候选线索输入
- 是面向 Codex、本地脚本，还是 OpenClaw / 龙虾优先：
  - 首版优先 Codex / 本地脚本
- 是否需要 `for-openclaw/` 变体：
  - 首版不必立即提供
  - 等输入结构稳定后再决定

## 课程与内容映射

- 对应课程：
  - 外贸 AI 落地营中的主动开发链路样板
- 可讲成什么案例：
  - “从搜索结果到可背调线索池”
  - “为什么搜索结果很多，但真正能进入开发动作的只有一部分”
- 可衍生什么内容选题：
  - 如何把公开搜索结果整理成销售能用的线索
  - 为什么线索初筛比盲目背调更省时间

## 验收方式

- 最小测试输入是什么：
  - 至少 3 条来源不一致、字段缺失程度不同的候选线索
- 预期输出应包含哪些关键信息：
  - 统一字段
  - 缺失项
  - 初步分类
  - 衔接客户背调的标准输入
  - 建议下一步动作
- 如何判断这个 Skill 已达到可演示：
  - 能把样例线索整理成结构化结果，并清楚展示哪些可直接进入客户背调
- 如何判断这个 Skill 已达到可交付：
  - 有固定输入结构、最小样例、固定输出结构、字段说明、风险边界说明，并能稳定产出兼容客户背调的标准输出
