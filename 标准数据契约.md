# 外贸主动开发标准数据契约

这份文档定义当前公开主线的统一数据契约。

核心原则只有三条：

- Skill 先产出标准字段，再谈写回哪种容器
- 飞书是课堂标准沙盘，不是企业唯一容器
- 没有通过质量门槛，就不要继续自动往下游推进

## 核心实体

### `LeadRecord`

一条主动开发线索的最小主记录。

最关键字段：

- `lead_id`
- `company_name`
- `person_name`
- `email`
- `country_or_market`
- `source_stage`
- `current_stage`
- `current_status`
- `recommended_next_action`

### `EvidenceItem`

一条公开证据。

最关键字段：

- `title`
- `url`
- `source_type`
- `note`
- `confidence`

### `ScreeningDecision`

线索整理阶段的判断结果。

最关键字段：

- `evidence_grade`
- `missing_fields`
- `manual_review_reasons`
- `recommended_next_action`
- `legacy_recommended_next_action`

### `IntelDecision`

客户背调阶段的是否放行判断。

最关键字段：

- `entity_confidence`
- `evidence_sufficiency`
- `risk_rating`
- `recommended_next_action`
- `manual_review_required`

### `OutreachDraftPackage`

复核型开发信工作台的输出包。

最关键字段：

- `subject_options`
- `drafts`
- `review_notes`
- `evidence_signals_used`
- `unconfirmed_fact_checklist`
- `send_policy`
- `workflow_guidance`

### `ContainerBundle`

同一轮主动开发产出的容器中立结果包。

默认派生输出：

- JSON
- Markdown
- CSV
- Feishu Sandbox Adapter

## 阶段动作枚举

### 搜索阶段

- `ready_for_screening`
- `needs_enrichment`
- `hold_for_manual_review`
- `reject_low_evidence`

### 线索整理阶段

- `ready_for_customer_intel`
- `needs_enrichment`
- `hold_for_manual_review`

兼容旧字段：

- `ready_for_customer_intel -> enter_customer_intel`
- `needs_enrichment -> enrich_then_customer_intel`

### 客户背调阶段

- `ready_for_email_draft`
- `hold_for_manual_review`

### 开发信阶段

- `ready_for_manual_send`
- `hold_for_manual_review`

固定发送策略：

- `send_policy = manual_review_only`

## 三层数据容器

### 第一层：保底容器

- JSON
- Markdown
- CSV

### 第二层：课堂标准沙盘

- Feishu Sandbox Adapter

### 第三层：企业真实容器

- CRM Adapter
- ERP Adapter
- 邮箱草稿箱 Adapter

当前公开仓库只公开前两层和第三层的接口预留。
